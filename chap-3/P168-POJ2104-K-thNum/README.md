# POJ2104 K-th num
n個の数字からなる数列が与えられる。クエリ(i, j k)に対して、a_i ~ a_jを整列させたk番目の数を答える問題。  
## バケットソート版
平方分割といって、全体がn個とするとsqrt(n)個ごとに分割すると計算量がなんかいい感じになるというものがある。  
(しかし今回は、各要素についての処理はO(1)で、バケットについての処理はO(log バケットの中身の個数)となるので、バケットの個数を減らして、バケットを増やすと良いらしいby蟻本。)  
  
バケットに分割し、各バケットをソートする。また、元の数列のソートしたものも別に持つ。  
どの数が、[i, j]でk番目になるのかを、元の数列全体をソートしたものから二分探索することで、クエリに答えられる。  
## セグメント木版
ノードが範囲を担当するのはセグメント木の通例と同じ。しかし今回は、ノードはその担当範囲をソートした配列を持つ。  
範囲に含まれるノードたちの中で、与えられた数字が何番目かを足し算していくと、[i,j]の中で与えられた数字が何番目なのかが求められる  

やはり、元の数列全体をソートした数列を二分探索して、[i,j]でk番目になる数を求めることでクエリに答えていく。
